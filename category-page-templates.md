# Category Page Templates

Choose a template by product category, then adapt the page count and emphasis to the user's brief.

## Universal conversion logic

1. attention: stop the scroll and make product identity clear
2. understanding: show what the product is and why it matters
3. desire: create scene, texture, lifestyle, or outcome appeal
4. trust: show details, specs, materials, process, or safe-use logic
5. action: close with purchase reason and next step

## Universal 8-page template

1. **封面主视觉**: product identity, emotional hook, primary benefit
2. **核心卖点**: 3-5 scan-friendly selling points
3. **细节/材质/原料**: texture, material, ingredients, or craft
4. **场景/人群共鸣**: target customer and usage moment
5. **使用方式/功能说明**: how to use, wear, apply, eat, clean, or install
6. **优势/选择理由**: why this product is worth choosing
7. **规格/包装/参数**: rational information and SKU clarity
8. **信任/成交收口**: summary, family use, daily use, purchase intent

## Food / beverage / tonic ingredient

Recommended sequence:

1. 封面主视觉: product + natural scene + warm daily-drinking hook
2. 核心卖点: natural material, easy brew, daily pairing, clean look
3. 原材品质: close-up texture, slice/particle/color/cleanliness if relevant
4. 日常饮用场景: home, office, outdoor rest, family table
5. 冲泡/搭配方式: cup, thermos, tea pot, recipe-style but simple
6. 细节展示: product form, packaging opening, portion suggestion if known
7. 包装规格: package, quantity, storage reminder if provided
8. 家庭常备/收口: warm lifestyle purchase reason

Safe copy angle: lifestyle, natural ingredient, daily ritual, convenience. Avoid medical/health function claims.

## Skincare / personal care

Recommended sequence:

1. 封面主视觉: product texture and clean beauty mood
2. 痛点共鸣: daily use discomfort or need, without medical fear
3. 核心卖点: texture, experience, convenience, routine fit
4. 成分/配方亮点: only user-provided ingredients or general texture cues
5. 使用肤感/场景: bathroom, vanity, travel, daily routine
6. 使用步骤: clear steps and amount if known
7. 包装规格: bottle/jar/tube size, portability, hygiene
8. 收口成交: routine completion and emotional value

## Home / cleaning

Recommended sequence:

1. 封面主视觉: clean home transformation mood
2. 痛点切入: daily mess or inconvenience
3. 核心功能: cleaning, storage, convenience, design
4. 场景适配: kitchen, bathroom, living room, bedroom
5. 使用步骤: simple before/during/after workflow
6. 对比优势: convenience and experience, not unsupported absolute claims
7. 细节规格: material, size, package, refill, compatibility
8. 收口成交: home upgrade and daily ease

## Apparel

Recommended sequence:

1. 封面主视觉: model/product mood and style identity
2. 上身/搭配氛围: target scenario and fashion emotion
3. 面料细节: texture, drape, breathability only if provided
4. 版型亮点: cut, silhouette, details
5. 多场景穿搭: commute, casual, travel, date, home
6. 颜色/尺码: SKU and sizing guidance
7. 工艺细节: stitching, buttons, waist, pocket, zipper
8. 收口成交: wardrobe value

## Electronics / accessories

Recommended sequence:

1. 封面主视觉: product and core use case
2. 核心功能: 3-5 key functions or specs provided by user
3. 痛点解决: cable clutter, battery anxiety, compatibility, portability
4. 参数亮点: power, size, material, interface, capacity if provided
5. 场景适配: office, travel, car, home, gaming
6. 兼容说明: devices and limits if known
7. 细节展示: ports, texture, buttons, package contents
8. 收口成交: practical daily upgrade

## Page count adaptation

- 6 pages: merge details + specs, merge trust + close.
- 7 pages: use universal sequence without separate trust page.
- 8 pages: default for most products.
- 9-10 pages: add pain point, comparison, FAQ, bundle/SKU, or proof page only when information is available.


## Category conversion formulas

Use these formulas before choosing the final page sequence. A formula is the category-specific purchase logic, not a fixed page list.

### Apparel conversion formula

fit/style anxiety -> on-body or silhouette proof -> fabric and comfort detail -> styling scenarios -> color/size decision -> craft details -> care/service -> wardrobe value

Useful pages:
- hero style identity
- target body/scene relevance
- on-body silhouette or flat-lay structure
- fabric texture and drape
- multi-scene styling
- size/color/SKU clarity
- detail craft and care
- soft closing wardrobe value

Risk controls:
- Do not promise slimming, body transformation, or universal fit.
- Do not invent fabric composition, model height/weight, or washing durability.

### Food / beverage conversion formula

appetite or daily ritual -> ingredient/form visible proof -> taste/texture expectation -> serving/use scenario -> package/portion clarity -> storage/logistics -> household/stock-up close

Risk controls:
- Do not write health treatment, physiological improvement, immunity, sleep, fatigue, liver/kidney, blood, dampness, fire, or medicine-like claims.
- If ingredient origin, grade, year, organic status, or testing is not provided, do not invent it.

### Skincare / personal care conversion formula

routine need -> texture/sensory proof -> provided ingredient or formula highlight -> use steps -> daily scenario -> package hygiene/portability -> suitable routine close

Risk controls:
- Do not promise treatment, cure, guaranteed repair, permanent results, 100% allergy-free, medical-grade outcome, or before/after cure.
- Use texture, routine, comfort, and daily care language.

### Home / cleaning conversion formula

home mess or inconvenience -> product solution -> function proof -> room-specific scenario -> usage steps -> material/size/package -> service/refill -> daily home upgrade close

Risk controls:
- Do not write 100% sterilization, permanent mold prevention, disease prevention, or health protection unless proof and category allow it.

### Electronics / accessories conversion formula

usage pain -> core spec/function -> compatibility boundary -> scenario proof -> interface/detail shot -> safety/durability if provided -> package contents -> practical daily upgrade close

Risk controls:
- Do not invent power, capacity, certification, compatibility, chipset, material, waterproof level, or battery performance.
- For compatibility, state only provided or common-but-clearly-qualified ranges.

### Beauty tools / small appliances conversion formula

usage problem -> product mechanism -> mode/function explanation -> detail/attachment proof -> routine scenario -> cleaning/storage -> specs/safety if provided -> convenience close

Risk controls:
- Do not imply medical, body-shaping, hair-regrowth, skin-treatment, or guaranteed results unless clearly supported and legally allowed.

### Pet products conversion formula

pet owner problem -> pet-friendly design -> material/size proof -> usage scenario -> cleaning/maintenance -> size/species suitability -> package/service -> daily care close

Risk controls:
- Do not promise medical, behavioral cure, absolute safety, or veterinarian endorsement without evidence.

### Gift products conversion formula

occasion emotion -> product meaning -> package/presentation -> personalization if provided -> usage or display scenario -> recipient fit -> delivery/service -> gifting close

Risk controls:
- Do not invent luxury status, official collaboration, famous designer, or cultural certification.

## Page-count logic by complexity

- 5-6 pages: simple, low-price, impulse products; merge proof and detail, merge specs and close.
- 7-8 pages: default for most mobile ecommerce products.
- 9-10 pages: higher price, high-risk, multi-SKU, technical, apparel sizing, or products needing more proof.
- 11+ pages: only when the user explicitly requests a long page or the product has real evidence, FAQ, comparison, and SKU complexity.

## High-conversion page sequence templates

### 6-page direct-response template

1. Hero and core benefit
2. Problem or usage relevance
3. Top 3 selling points
4. Visible proof or detail
5. Specs/package/usage clarity
6. Offer/service/action close

### 8-page balanced template

1. Hero and click validation
2. Buyer relevance or pain point
3. Core value points
4. Detail/material/spec proof
5. Usage or lifestyle scenario
6. Differentiation or choosing reason
7. SKU/specs/risk reduction
8. Trust/action close

### 10-page considered-purchase template

1. Hero and core promise
2. Buyer problem and relevance
3. Solution overview
4. Feature-benefit breakdown
5. Evidence/detail proof
6. Usage workflow or scene
7. Comparison or choosing guide
8. Specs/SKU/compatibility
9. Trust/service/FAQ
10. Offer/action close
