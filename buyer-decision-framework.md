# Buyer Decision Framework

Use this framework before page planning. A detail page should not merely list features; it should reduce buyer uncertainty step by step.

## 9-stage mobile detail-page decision path

1. **click validation / traffic match**
   - Buyer question: Did I land on the product I expected?
   - Page job: show the product, category, primary promise, and visual identity immediately.
   - Failure mode: beautiful cover image but unclear product or mismatch with ad/search intent.

2. **personal relevance**
   - Buyer question: Is this for my situation, need, body type, home, device, skin, taste, or use case?
   - Page job: connect the product to target audience, scenario, or pain point.
   - Failure mode: generic brand language that does not mirror user intent.

3. **core value**
   - Buyer question: Why should I care?
   - Page job: state the top 3-5 purchase reasons in scan-friendly form.
   - Failure mode: feature dumping without a clear benefit hierarchy.

4. **differentiation**
   - Buyer question: Why choose this instead of another option or doing nothing?
   - Page job: show the practical difference, upgrade, convenience, design, texture, bundle, or fit.
   - Failure mode: unsupported claims such as best, leading, top, permanent, guaranteed.

5. **proof and tangibility**
   - Buyer question: Can I see evidence?
   - Page job: use product details, macro shots, real materials, parameters, visible structure, provided test reports, provided reviews, or provided credentials.
   - Failure mode: claims without proof, fake badges, fake certificates, fake reviews.

6. **usage imagination**
   - Buyer question: How will I use it in real life?
   - Page job: show scenario, routine, usage steps, styling, installation, serving, cleaning, storage, or workflow.
   - Failure mode: isolated product posters with no usage context.

7. **risk reduction**
   - Buyer question: What could go wrong, and how do I avoid buying the wrong thing?
   - Page job: clarify compatibility, size, fit, SKU, amount, usage limits, storage, care, installation, shipping, or service.
   - Failure mode: missing specifications, unclear scale, no compatibility boundaries.

8. **trust reinforcement**
   - Buyer question: Can I trust this seller/product?
   - Page job: present only verified trust assets: provided qualifications, warranty, service policy, real review assets, package clarity, process facts, official store facts.
   - Failure mode: invented authority, expert endorsement, platform logos, awards, rankings, sales numbers.

9. **action close**
   - Buyer question: Should I buy now, and which option should I choose?
   - Page job: summarize product value, SKU/bundle, offer, service, and soft CTA.
   - Failure mode: no clear closing reason or excessive pressure.

## Decision-stage tags

Use one tag per page:

- `attention`: stop the scroll and validate the click.
- `relevance`: connect with target need, pain point, or scenario.
- `value`: explain the main purchase reason.
- `differentiation`: explain why this option is worth choosing.
- `proof`: support a claim with visible or provided evidence.
- `usage`: help the buyer imagine real use.
- `risk_reduction`: reduce wrong-purchase anxiety.
- `trust`: show verified seller/product confidence cues.
- `action`: close with a buying reason or next step.

## Page planning rule

Each page must answer exactly one primary buyer question. It may carry secondary information, but the primary message must be clear at phone-screen size.

For most 8-page pages, use:

1. attention / click validation
2. relevance / pain point or use case
3. value / core selling points
4. proof / material, structure, ingredient, parameter, or detail
5. usage / scenario or instructions
6. differentiation / choosing reason or comparison with old habit
7. risk_reduction / specs, SKU, compatibility, size, care, package
8. trust or action / service, provided proof, value summary, soft CTA

## First-three-page rule

The first three pages must answer:

1. What is this product?
2. Why is it relevant to me?
3. Why is it worth continuing or buying?

If these are not answered, the page sequence is weak even if later pages are attractive.

## Price and involvement adaptation

- Low price / impulse purchase: shorten proof, increase scene and offer clarity, keep text very direct.
- Mid price: balance emotional appeal, practical features, details, and service.
- High price: increase evidence, specifications, comparison, warranty, process, material proof, and buyer risk reduction.
- Standardized commodity: emphasize price-value, specs, compatibility, bundle, and convenience.
- Non-standard aesthetic product: emphasize style, scenario, texture, fit, mood, and visual identity.
- High-risk regulated product: emphasize safe lifestyle language, instructions, and verified information only.
