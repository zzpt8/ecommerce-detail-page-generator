# 卖点证据规则

在合规画像和文案撰写之前使用。本文件目标是阻止无证据主张，同时通过可见证据增强说服力。

## 主张类型

将每个卖点分类为以下一种或多种：

- `feature`：产品有什么或是什么。
- `benefit`：买家从功能中获得什么。
- `experience`：感官、生活方式、便利或情绪体验。
- `proof`：证书、检测、评价、数据、流程、产地、奖项或官方事实。
- `offer`：价格、组合、赠品、包邮、售后、保障。
- `comparison`：与替代品、旧习惯、普通产品或 SKU 选择的对比。
- `trust`：服务、店铺、品牌、资质、买家证明、生产证明。

## 主张强度

- `strong`：精确、可测量、官方、对比、保证、医疗、功能性或具体性能主张。
- `moderate`：合理但未量化的功能-利益关系。
- `soft`：场景、生活方式、质感、情绪、便利、日常或偏好表达。

## 证据来源等级

- `user_provided`：用户明确提供。
- `visible_product_proof`：从上传产品图或可见结构/材质/包装可观察。
- `derived_from_category`：谨慎类目推断，必须保持软性。
- `unsupported`：无来源，不能写成事实。

## 证据门槛

对每个主张执行：

1. 强主张若没有用户提供的证据，删除或降级。
2. 认证、检测报告、奖项、排名、产地、销量、明星、医生、机构、政府、平台或专家背书，必须有明确用户证据。
3. 仅从类目推断的主张，只能写成软性场景或可能性，不能写成事实。
4. 产品图可见的内容，只描述“看得见的部分”，不要发明原因或性能。
5. 重要但无证据的主张，创建证明槽位或标记为缺失证据，不进入客户文案。

## 安全降级模式

- 精确功效 → 日常体验或生活方式语言。
- 医疗/生理改善 → 生活陪伴或普通使用场景。
- 最好/第一/顶级 → 合适选择、实用升级、贴心细节。
- 保证结果 → 设计用于支持、帮助创造、更容易、更方便。
- 未知产地/工艺 → 可见材质质感、干净呈现、细节特写。
- 未知销量/评价 → 删除，不伪造社会证明。
- 未知认证 → 删除，不生成徽章视觉。

## 映射字段

每个主张记录：

- `claim`：原始或拟定主张。
- `claim_type`：feature、benefit、experience、proof、offer、comparison 或 trust。
- `source`：user_provided、visible_product_proof、derived_from_category 或 unsupported。
- `claim_strength`：strong、moderate 或 soft。
- `evidence_required`：true / false。
- `evidence_available`：true / false。
- `allowed_customer_copy`：最终页面可用安全文案。
- `visual_proof`：支持它的图片、细节或场景。
- `risk_note`：需要规避的表达。

## 视觉证明规则

好的视觉证明：

- 材质主张：材质/纹理特写。
- 包装主张：开箱或数量平铺。
- 电子兼容性：接口/按钮/端口细节。
- 尺寸主张：比例参照。
- 使用主张：步骤化场景。
- 信任主张：用户提供的报告/照片/评价，不篡改、不夸大。

差的视觉证明：

- 假印章、假奖牌、假官方标识。
- 普通滋补/日用产品使用医生、医院、医疗图标。
- 未验证功效的前后对比。
- 夸张攻击竞争对手。
- 伪造买家截图或评价卡片。

## 文案决策示例

- 用户说材质为 316 不锈钢：可以写 316 不锈钢，并展示细节证明。
- 用户说“质量好”：只能写质感/细节软性表达，不写量化质量证明。
- 用户说销量 10 万：仅在用户提供且平台/类目允许时可用，否则标记需证据。
- 用户未提供产地：不要写源头产地、产地直发、进口、精选某地、农场直供。
- 用户未提供报告：不要创建检测报告视觉、质检印章或证书徽章。
