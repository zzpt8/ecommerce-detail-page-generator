# 买家决策框架

在页面规划之前使用本框架。详情页不应只是罗列功能，而应一步步降低买家的不确定性。

## 9 阶段移动端详情页决策路径

1. **点击验证 / 流量承接**
   - 买家问题：我点进来的是否是我期待的产品？
   - 页面任务：立即展示产品、类目、主承诺和视觉身份。
   - 失败模式：封面好看但看不出产品，或与广告/搜索意图不匹配。

2. **个人相关性**
   - 买家问题：这是否适合我的场景、需求、身体、家庭、设备、肤质、口味或用法？
   - 页面任务：将产品连接到目标人群、场景或痛点。
   - 失败模式：泛泛品牌语言，没有回应用户意图。

3. **核心价值**
   - 买家问题：为什么我要在意？
   - 页面任务：用易扫读方式说明 3-5 个核心购买理由。
   - 失败模式：只堆功能，没有利益优先级。

4. **差异化**
   - 买家问题：为什么选择它，而不是其他产品或不买？
   - 页面任务：展示实际差异、升级点、便利性、设计、质感、组合或匹配度。
   - 失败模式：无证据写“最好、领先、顶级、永久、保证”等。

5. **证明与可感知性**
   - 买家问题：我能看到证据吗？
   - 页面任务：用产品细节、材质特写、参数、结构、用户提供的报告/评价/资质来支持主张。
   - 失败模式：主张无证据、假徽章、假证书、假评价。

6. **使用想象**
   - 买家问题：我真实生活中怎么用？
   - 页面任务：展示场景、日常、步骤、搭配、安装、清洁、收纳或工作流。
   - 失败模式：只有孤立产品海报，没有使用语境。

7. **风险降低**
   - 买家问题：可能哪里买错？如何避免？
   - 页面任务：明确兼容性、尺寸、适用边界、SKU、用量、使用限制、储存、护理、安装、物流或服务。
   - 失败模式：缺规格、尺度不清、兼容边界缺失。

8. **信任强化**
   - 买家问题：我能信任这个卖家/产品吗？
   - 页面任务：只展示已验证信任资产，如用户提供的资质、保修、服务政策、真实评价素材、包装清单、流程事实或官方店事实。
   - 失败模式：虚构权威、专家背书、平台 logo、奖项、排名、销量。

9. **行动收口**
   - 买家问题：我现在是否该买？选择哪个规格？
   - 页面任务：总结价值、SKU/组合、优惠、服务和温和 CTA。
   - 失败模式：没有清晰收口理由，或过度施压。

## 决策阶段标签

每页使用一个标签：

- `attention`：停止滚动，验证点击。
- `relevance`：连接目标需求、痛点或场景。
- `value`：解释主要购买理由。
- `differentiation`：解释为什么值得选它。
- `proof`：用可见或用户提供的证据支持主张。
- `usage`：帮助买家想象真实使用。
- `risk_reduction`：降低买错焦虑。
- `trust`：展示可验证信心线索。
- `action`：用购买理由或下一步收口。

## 页面规划规则

每页必须回答一个主要买家问题。可以承载辅助信息，但手机屏幕尺寸下必须清楚主信息。

大多数 8 页详情页使用：

1. attention / 点击验证
2. relevance / 痛点或使用场景
3. value / 核心卖点
4. proof / 材质、结构、成分、参数或细节
5. usage / 场景或说明
6. differentiation / 选择理由或与旧习惯对比
7. risk_reduction / 规格、SKU、兼容性、尺寸、护理、包装
8. trust 或 action / 服务、已提供证明、价值总结、温和 CTA

## 前三页规则

前三页必须回答：

1. 这是什么产品？
2. 为什么与我相关？
3. 为什么值得继续看或购买？

如果这些没有回答，即使后面的页面很好看，详情页序列也偏弱。

## 价格与决策复杂度适配

- 低价/冲动购买：缩短证明，增加场景和优惠清晰度，文字更直接。
- 中等客单：平衡情绪、实用功能、细节和服务。
- 高客单：增加证据、规格、对比、保修、流程、材质证明和风险降低。
- 标品：强调价格价值、规格、兼容性、组合和便利。
- 非标审美产品：强调风格、场景、质感、匹配、氛围和视觉身份。
- 高风险监管产品：强调安全生活方式语言、使用说明和已验证信息。
